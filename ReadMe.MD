# DIMS Dashboard - Client-Side Setup Guide

Github website: https://wimpouw.github.io/DIMS_dashboard/

## File Structure

```
/your-project/
├── index.html                    # Main HTML file
├── config.json                   # Dataset configuration
├── css/
│   └── reset.css                 # Dashboard styles
├── js/
│   ├── app.js                    # Main application logic
│   └── video-component.js        # Enhanced video component
├── optional_requirements.txt     # Python deps for optional analyses
├── optional_step_RQA.py          # RQA pre-computation script
├── optional_step_crosswavelet.py # Cross-wavelet pre-computation script
└── assets/
    ├── timeseries/
    │   └── {videoID}_{dataType}.csv
    ├── transcripts/
    │   └── {videoID}_transcript.json
    ├── videos/
    │   └── {videoID}.mp4
    ├── rqa/                       # generated by optional_step_RQA.py
    │   └── {videoID}_rqa_data.json
    ├── crosswavelet/              # generated by optional_step_crosswavelet.py
    │   └── {videoID}_crosswavelet_data.json
    └── elan/                      # optional — ELAN annotation files
        └── {videoID}.eaf
```

## Files You Need to Create

### 1. Save the artifacts as files:
- `index.html` - The main HTML file (now uses external CSS)
- `config.json` - Configuration file
- `css/styles.css` - Dashboard styles (extracted from HTML)
- `js/app.js` - Main application logic (simplified for single-column CSVs)
- `js/video-component.js` - Enhanced video component with reliable time seeking

## Optional Features

### RQA (Recurrence Quantification Analysis)
1. Add `"include_RQA": ["dataType1", ...]` to `config.json`
2. Run `python optional_step_RQA.py` — outputs JSON to `assets/rqa/`

### Cross-Wavelet Analysis
1. Add `"include_crosswavelet": ["dataType1", "dataType2"]` to `config.json`
2. Run `python optional_step_crosswavelet.py` — outputs JSON to `assets/crosswavelet/`

### ELAN Annotations
1. Add `"include_elan": true` to `config.json`
2. Place `.eaf` files in `assets/elan/{videoID}.eaf`
   - ELAN files must use `ALIGNABLE_ANNOTATION` elements with `TIME_SLOT` references
   - The ELAN Annotations tab appears automatically when enabled and a `.eaf` file is found
   - Each tier is color-coded and can be toggled with checkboxes
   - Clicking any annotation seeks the video to that time point


### 2. Use your existing reset.css:
Place your existing `reset.css` in the `css/` folder.

## Simplified Data File Requirements

### CSV Files (in `assets/timeseries/`)
**Each CSV should contain EXACTLY:**
- `Time` column (required for merging)
- **One measurement column** (any name you want)

Example CSV structure:
```csv
Time,synchrony_measure
0.0,0.5
0.1,0.6
0.2,0.4
...
```

### Configuration Example
```json
{
    "videoIDs": ["3120"],
    "dataTypes": {
        "3120": ["syncbody", "neuralsync", "neuralsync2", "neuralsync3"]
    },
    "defaultWindowSize": 5,
    "title": "DIMS Dashboard",
    "subtitle": "Multimodal Synchrony Exploration Visualization",
    "authors": "Grace Qiyuan Miao, James Trujillo, Landry S. Bulls, Mark A. Thornton, Rick Dale, Wim Pouw",
    "contacts": "contacts: q.miao@ucla.edu & w.pouw@tilburguniversity.edu"
}
```

This will create 4 separate subplots:
- `syncbody_synchrony_measure`
- `neuralsync_synchrony_measure`  
- `neuralsync2_synchrony_measure`
- `neuralsync3_synchrony_measure`
