# DIMS Dashboard

**Authors:** Grace Qiyuan Miao, Wim Pouw, Michal Weiss

An open tool for visualizing multimodal social interaction data — time series, video, transcripts, and optional analyses (RQA, cross-wavelet, ELAN annotations).

GitHub: https://github.com/mikub97/DIMS_dashboard

## File Structure

```
/your-project/
├── index.html                 # Main HTML file
├── config.json                # Dataset configuration
├── css/
│   └── reset.css              # Dashboard styles
├── js/
│   ├── app.js                 # Main application logic
│   └── video-component.js     # Enhanced video component
├── opt/                       # Optional pre-computation scripts
│   ├── requirements.txt       # Python deps for optional analyses
│   ├── step_RQA.py            # RQA pre-computation script
│   └── step_crosswavelet.py   # Cross-wavelet pre-computation script
└── assets/
    ├── timeseries/
    │   └── {videoID}_{dataType}.csv
    ├── transcripts/
    │   └── {videoID}_transcript.json
    ├── videos/
    │   └── {videoID}.mp4
    ├── rqa/                   # generated by opt/step_RQA.py
    │   └── {videoID}_rqa_data.json
    ├── crosswavelet/          # generated by opt/step_crosswavelet.py
    │   └── {videoID}_crosswavelet_data.json
    └── elan/                  # optional — ELAN annotation files
        └── {videoID}.eaf
```

## Optional Features

Install Python dependencies once: `pip install -r opt/requirements.txt`

### RQA (Recurrence Quantification Analysis)
1. Add `"include_RQA": ["dataType1", ...]` to `config.json`
2. Run `python opt/step_RQA.py --config config.json --output-dir assets/rqa`

### Cross-Wavelet Analysis
1. Add `"include_crosswavelet": ["dataType1", "dataType2"]` to `config.json`
2. Run `python opt/step_crosswavelet.py --config config.json --output-dir assets/crosswavelet`

### ELAN Annotations
1. Add `"include_elan": true` to `config.json`
2. Place `.eaf` files in `assets/elan/{videoID}.eaf`
   - ELAN files must use `ALIGNABLE_ANNOTATION` elements with `TIME_SLOT` references
   - The ELAN Annotations tab appears automatically when enabled and a `.eaf` file is found
   - Each tier is color-coded and can be toggled with checkboxes
   - Clicking any annotation seeks the video to that time point


## Data File Requirements

### CSV Files (in `assets/timeseries/`)
**Each CSV should contain EXACTLY:**
- `Time` column (required for merging)
- **One measurement column** (any name you want)

Example CSV structure:
```csv
Time,synchrony_measure
0.0,0.5
0.1,0.6
0.2,0.4
...
```

### Configuration Example
```json
{
    "videoIDs": ["3120"],
    "dataTypes": {
        "3120": ["syncbody", "neuralsync", "neuralsync2", "neuralsync3"]
    },
    "defaultWindowSize": 5,
    "title": "DIMS Dashboard",
    "subtitle": "Multimodal Synchrony Exploration Visualization",
    "authors": "Your Name, Collaborator Name",
    "contacts": "contacts: your@email.com"
}
```

This will create 4 separate subplots:
- `syncbody_synchrony_measure`
- `neuralsync_synchrony_measure`  
- `neuralsync2_synchrony_measure`
- `neuralsync3_synchrony_measure`

## Local Development

Use the included server (plain `python -m http.server` does not support HTTP Range requests, which breaks video seeking):

```bash
python serve.py        # default port 8000
python serve.py 3000   # custom port
```

## Deployment

Deploy to any static hosting service — all of these support HTTP Range requests for video seeking:
- **GitHub Pages**
- **Netlify**
- **Vercel**
- **Any standard web server**
